GOLDEN RULES (read these first)
  1. Ship a mock before you ship the real thing. No mock = you block everyone downstream.
  2. Your tool is also a target. Every team builds an admin page with security-event tracking, separate from its normal operational page.
  3. One schema to rule them all. Extra fields are fine; missing required fields are not.
  4. Don't fetch secrets into Claude prompts. Use env vars and reference the variable name, never the value.
  5. Health endpoint on everything. GET /health → 200 with your tool name.
  6. Graceful-degrade on bad input. Log it, reject it, return 400, keep running.
  7. Commit often. Every 30 minutes, even if broken. Git is your undo button.
