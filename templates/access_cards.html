GIT & PUSH — NO GITHUB ACCOUNT NEEDED
Your team's Lightsail has a deploy key pre-installed, pre-configured for SSH, and scoped to exactly your team's repo. No personal GitHub account, no public key, no private key to copy. Just SSH into your Lightsail and use git normally — clone, commit, push — no prompts, no passwords.
CLONE YOUR REPO (ON YOUR LIGHTSAIL)
# Team 1 — Trap
git clone git@github.com:sanctum-sec/soc-day3-trap.git

# Team 2 — Scout
git clone git@github.com:sanctum-sec/soc-day3-scout.git

# Team 3 — Analyst
git clone git@github.com:sanctum-sec/soc-day3-analyst.git

# Team 4 — Hunter
git clone git@github.com:sanctum-sec/soc-day3-hunter.git

# Team 5 — Dispatcher
git clone git@github.com:sanctum-sec/soc-day3-dispatcher.git

# Team 6 — Inspector
git clone git@github.com:sanctum-sec/soc-day3-inspector.git
COMMIT & PUSH (NORMAL GIT)
cd soc-day3-<team>

# Personalize your commits so teammates can see who did what
git config user.name "Olha"
git config user.email "olha@workshop.local"

# Work, then ...
git add .
git commit -m "add honeypot HTTP endpoint"
git push
The Lightsail's deploy key only has write access to your team's repo. You literally cannot push to another team's repo by accident from your box — GitHub rejects the auth.
Ukrainian character support: git config accepts UTF-8. Commit messages in Ukrainian work fine — the commit log, GitHub UI, and everyone's git log will render them correctly.
PAT ALTERNATIVE — FOR LAPTOP PUSH (OPTIONAL)
If you really want to push from your laptop instead of the Lightsail, ask the instructor for a Personal Access Token (PAT). Creating one takes ~2 minutes in the GitHub web UI; the instructor will paste the token here on the access-cards page when ready. Then clone with: 
git clone https://<PAT>@github.com/sanctum-sec/soc-day3-<team>.git
# or if you already cloned:
git remote set-url origin https://<PAT>@github.com/sanctum-sec/soc-day3-<team>.git
The default — work on your Lightsail — is simpler, faster, and the way everything else on Day 3 assumes you're set up.
CLAUDE CODE — ALREADY AUTHENTICATED
On every Lightsail above, Claude Code is pre-authenticated — just type claude and it works. The shared SOC-protocol bearer token is pre-loaded in ~/.soc_env and sourced on login.

If you installed Claude Code on your own laptop during Day 1, paste one line into your shell config after the instructor shares the API key. Claude Code picks it up silently — no prompt, no UI.
MAC — TERMINAL (~/.zshrc)
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.zshrc
source ~/.zshrc
WINDOWS — GIT BASH (~/.bashrc)
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.bashrc
source ~/.bashrc
Then run claude — authenticated, no further setup. Replace sk-ant-... with the API key the instructor shares.
